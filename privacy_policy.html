隐私政策

生效日期: 2026年4月16日

1. 开发者信息

  • 应用名称: 我的植物收集册 (MyPlantBook)
  • 开发者: 个人开发者
  • 联系邮箱: myplantbook@163.com

2. 我们收集的数据

2.1 您主动提供的数据

数据类型 用途 存储位置
植物照片 植物识别 上传至 ImgBB（有损压缩图）和七牛云（高清原图）用于识别处理
地理位置 1. 记录照片拍摄位置，用于地图标记
2. Trail 功能实时定位，用于路径记录和导航		 照片位置：与识别记录关联存储

用户标识 Supabase 生成的唯一用户 ID（UUID），用于数据关联 Supabase 数据库

2.2 自动收集的技术数据

  • 无设备信息收集
  • 无广告追踪标识符
  • 无第三方 Analytics 工具

3. 数据存储与处理

3.1 服务器位置

  • 应用服务器: Google Cloud Platform (asia-east2 区域)
  • 用户认证与数据库: Supabase
  • 植物识别数据库: NeonDB
  • 图片存储: ImgBB + 七牛云

3.2 数据跨境传输

您的个人信息可能存储于中国大陆境外（东北亚地区，如日本、韩国等地的云服务器）。我们已采取符合《中华人民共和国个人信息保护法》要求的安全措施保护您的数据，包括加密传输和存储。您继续使用本服务即视为知悉并同意上述数据跨境传输安排。

3.3 数据保留期限

数据类型 保留策略
植物照片 识别完成后保留，用于展示用户历史记录
识别结果 与照片关联保留
地理位置 照片位置：与识别记录关联保留
Trail 实时定位：仅实时使用，关闭功能后清除
账户信息 账户存续期间保留

4. 账户删除与数据权利

4.1 您拥有的权利

  • 访问权: 查看您上传的所有植物记录
  • 删除权: 删除您的账户及关联数据
  • 导出权: 您的数据存储在本地 CoreData 中，可通过设备备份导出

4.2 账户删除流程

在【账户管理】页面发起删除：

  1. 三层确认弹窗防止误操作
  2. 密码二次验证身份
  3. 数据软删除（user_id 置为 'USER_DELETED'）
  4. Supabase 用户身份删除
  5. 审计日志记录删除操作（用于安全审计）
  6. 本地数据清理

注意: 软删除后，您的识别记录将匿名化，无法关联到具体用户。

6. 用户权利与权限控制

6.1 权限均为可选

所有系统权限均为可选，您可以在首次使用时选择允许或拒绝，也可以随时在 iOS 系统设置中更改：

权限 用户拒绝后的影响 替代方案
相机 无法拍照识别 可从相册选择已有照片
相册 无法选择已有照片 可使用相机拍照
位置（照片） 照片不记录位置信息 正常识别，地图不显示该标记
位置（Trail） 无法记录路径和导航 Trail 功能受限，其他功能正常
网络 无法上传识别 需连接网络后使用

6.2 随时更改权限

设置路径：iOS 设置 → MyPlantBook → 权限管理

您可以随时开启或关闭任何权限，不会影响已保存的数据。

7. 数据安全

  • 所有数据传输使用 HTTPS/TLS 加密
  • 用户密码通过 bcrypt 加密存储（Supabase 处理）
  • 后端 API 使用 JWT 认证
  • 敏感操作（如删除账户）需要密码二次验证

7. 儿童隐私

本应用不面向 13 岁以下儿童。我们不会故意收集儿童的个人信息。

8. 政策更新

我们可能会更新本隐私政策。重大变更将通过应用内通知或邮件告知。

10. 权限说明

应用需要以下系统权限：

  • 相机: 拍摄植物照片进行识别
  • 照片库: 选择已有照片进行识别
  • 位置: 记录拍摄地点（可选，可在系统设置中关闭）
  • 网络: 上传照片进行云端识别

本隐私政策适用于 iOS 应用「我的植物收集册」(MyPlantBook)
