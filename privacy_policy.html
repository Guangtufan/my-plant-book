3.2 数据跨境传输

您的个人信息可能存储于中国大陆境外，包括：

  • 中国香港特别行政区：应用主服务器（Google Cloud asia-east2）
  • 法国：植物识别时，图片会传输至 Pl@ntNet 服务器进行处理

我们已采取符合《中华人民共和国个人信息保护法》要求的安全措施保护您的数据，包括加密传输和存储。在首次使用涉及跨境数据传输的功能（如植物识别）前，我们将通过应用内弹窗请求您的明确授权。如您不同意跨境传输，可选择不使用相关功能。

4. 账户删除与数据权利

4.1 您拥有的权利

  • 访问权：查看您上传的所有植物记录
  • 删除权：删除您的账户及关联数据
  • 导出权：您的数据主要存储于云端数据库（Supabase / NeonDB），您可通过应用内账户管理功能访问您的云端数据；本地 Core Data 仅作为缓存层用于离线访问，您也可通过设备备份导出本地缓存数据

4.2 账户删除流程

在【账户管理】页面发起删除：

  1. 三层确认弹窗防止误操作
  2. 身份二次验证（根据登录方式，可能需要密码或 Apple ID 验证）
  3. 数据软删除（user_id 置为 'USER_DELETED'）
  4. Supabase 用户身份删除
  5. 审计日志记录删除操作（用于安全审计）
  6. 本地数据清理

注意：软删除后，您的识别记录将与用户身份永久解除关联（匿名化），并在后续周期内从服务器物理删除。同时，本地数据也会被同步清理，确保云端与本地数据双向一致删除。

关于图片数据来源：应用内展示的植物参考图片来源于 Pl@ntNet 社区贡献，受 Creative Commons 等开源许可协议约束。这些图片仅用于应用内展示，不用于商业再分发。我们在应用内提供了详细的来源归属及许可信息展示。

关于数据处理责任：Pl@ntNet 仅作为植物识别目的的独立数据处理者（Data Processor），我们（应用开发者）仍是您个人数据的控制者（Data Controller）。Pl@ntNet 不会将您的图片用于其自身目的，也不会与其他方共享。

关于第三方服务使用原则：我们不会使用任何第三方服务进行用户追踪、广告定向或用户画像分析。所有第三方服务仅用于实现应用的核心功能。

6. 用户权利与权限控制

6.1 权限均为可选

所有系统权限均为 可选，您可以在首次使用时选择允许或拒绝，也可以随时在 iOS 系统设置中更改：

权限 用户拒绝后的影响 替代方案
相机 无法拍照识别 可从相册选择已有照片
相册 无法选择已有照片 可使用相机拍照

6.2 随时更改权限

设置路径：iOS 设置 → MyPlantBook → 权限管理

您可以随时开启或关闭任何权限，不会影响已保存的数据。

7. 应用内购买

本应用提供应用内订阅服务，由 Apple App Store 统一处理。我们不直接收集、存储或处理您的支付信息（如信用卡号、账单地址等）。所有订阅管理、付款处理及退款事宜均由 Apple 负责。

8. 数据安全

  • 后端 API 使用 JWT 认证
